# Skills Catalog — Available Specialized Capabilities

**Version:** 1.0  
**Last updated:** April 2026

The skills listed below are **already installed and available** in this project.

Before any implementation, always identify the required skills, declare them and justify their use.

## 🎯 Frontend & UI

- **vercel-react-best-practices**  
  → Official Vercel performance optimization guide for React and Next.js (69 prioritized rules in 8 categories).  
  **When to use:** Writing new components, data fetching, performance reviews, refactoring, bundle optimization.

- **frontend-design**  
  → Creation of distinctive, memorable production-grade interfaces with strong aesthetic direction.  
  **When to use:** When a unique visual identity is required.

- **shadcn**  
  → Building and maintaining UI components via CLI with full lifecycle management.  
  **When to use:** Consistent UI component work in Tailwind projects.

- **tailwind-design-system**  
  → Creation and maintenance of complete design systems with Tailwind CSS.  
  **When to use:** Defining or evolving visual systems and design tokens.

## 🔎 Discovery

- **find-skills**  
  → Skill for discovering new, high-quality skills from the skills.sh ecosystem to complement existing ones.  
  **When to use:** Only when current skills are clearly insufficient for the task or project needs.  
  **How to use:**  
  1. First evaluate if existing skills are sufficient.  
  2. If a gap exists, recommend specific new skills with:  
     - Skill name  
     - Brief description  
     - Clear justification (why it adds value to this task/project)  
     - Expected impact  
  3. Always ask for user confirmation before suggesting installation.  

  Do not recommend new skills unnecessarily. Keep the skill set lean and relevant.
