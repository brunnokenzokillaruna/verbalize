# Git Workflow Rules

**Version:** 1.0  
**Last updated:** April 2026

## 1. Atomic Commits (Mandatory)
- One commit per completed task or mini-task
- Commits must be atomic (single responsibility)

## 2. Commit Messages
- Use semantic Conventional Commits
- Always use the `git-commit` skill to generate standardized messages

## 3. Task Breakdown (Mandatory)
When implementing, removing or changing something significant:
- Break the work into smaller, precise **mini-tasks**
- Treat each mini-task as an atomic unit with its own commit

**End of Git Workflow Rules**
