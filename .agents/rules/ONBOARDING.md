# Onboarding Existing Projects

**Version:** 1.0  
**Last updated:** April 2026

When applying Core Rules to an ongoing project:

1. Create a branch `chore/core-rules-onboarding`
2. Run a full audit using the `audit-website` skill
3. Identify and document technical debt (files > 500 lines, hardcode, missing tests, accessibility issues, etc.)
4. Create issues for each debt item with priority
5. Apply the rules gradually (never all at once)
6. Update the project README with the Core Rules adoption date
